# Support

## Getting Help

Use the repository issue tracker for:

- bug reports
- feature requests
- documentation improvements

For general project questions:

- start with the [Quick Start](docs/QUICKSTART.md)
- read the [User Guide](docs/USER_GUIDE.md)
- walk through the [Tutorial](docs/TUTORIAL.md)

## What To Include

When asking for help, include:

- your operating system
- Rust version if relevant
- Qt version if relevant
- the command you ran
- the exact error message
- whether the problem is in the desktop, CLI, or backend

## Response Expectations

Parallax is an early-stage project, so support is best-effort. Clear, reproducible reports will get the fastest traction.
